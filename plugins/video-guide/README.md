# Video Guide Plugin

Create professional video guides with TTS narration, screen recording, and YouTube upload.

## Container Requirements

This plugin delegates all rendering to a `video-guide-creator` Docker container. The container must provide:

| Component | Purpose |
|-----------|---------|
| Edge TTS | Microsoft neural text-to-speech (requires internet) |
| FFmpeg | Video encoding and muxing |
| Playwright + Chromium | Headless browser for screencast recording |
| Python runtime | Orchestrator that parses YAML guides and drives rendering |

The container project lives at `/mnt/data/ai/video-guide-creator/` with this layout:

```
video-guide-creator/
├── brands/        # Brand presets (colors, voice, language, logo)
├── guides/        # YAML guide definitions (generated by this plugin)
├── output/        # Rendered MP4 files
├── auth/          # Playwright auth state for authenticated screencasts
├── credentials/   # YouTube API OAuth credentials
├── src/           # Container source code
└── docker-compose.yml
```

Build the container (one-time):

```bash
docker compose -f /mnt/data/ai/video-guide-creator/docker-compose.yml build
```

## Usage

```
/video-guide How to create an annual report
```

Or naturally: "Make a video guide about..."

The plugin can be invoked from any repository. Frontend code research uses the current working directory; all container operations target the centralized path above.

## Components

- **Skill** (`video-guide-creator`): Detects video guide requests, generates YAML, validates, renders via container
- **Command** (`/video-guide`): Direct invocation with topic argument — delegates to the skill

## Dependencies

- Docker Engine with docker compose v2+
- `video-guide-creator` container image (built from project above)
- Internet access for Edge TTS API calls
- YouTube API OAuth credentials (optional, for upload feature)
